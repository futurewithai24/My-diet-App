# 家計簿アプリ 連携仕様書

## 1. 文書概要
- 文書名: 家計簿アプリ 連携仕様書
- 対象システム: 家族収支管理アプリ（LIFF Webアプリ）
- 目的: Vercel / GitHub / LINE / Googleスプレッドシート / GAS の連携構成と、主要な機能・データフローを明文化する
- 作成日: 2026-04-22

## 2. システム構成（全体像）
- フロントエンド: `index.html`（単一HTML + JavaScript）
- 配信基盤: Vercel（GitHub連携デプロイ）
- 認証基盤: LINE LIFF（LINEログイン・プロフィール取得）
- バックエンドAPI: Google Apps Script（Webアプリ公開URL）
- データストア: Googleスプレッドシート
- ソース管理: GitHub リポジトリ `futurewithai24/My-budget-App`

## 4. 機能仕様（画面別）

### 4.1 入力タブ
- 入出金切替:
  - 出金 / 入金 の2種
- 入力項目:
  - カテゴリ
  - 金額
  - 内容/支払先
  - 日付
  - 写真（最大3枚）
- 写真アップロード:
  - カメラ撮影 or ギャラリー選択
  - Base64化して `imageBlobs` 配列へ保持
- 登録処理:
  - `action: "upload"` でGASへ送信
  - 成功時: 完了アラート後に画面再読み込み

### 4.2 履歴タブ
- フィルタ:
  - ユーザー（全員/個別）
  - 分類（カテゴリ）
  - 年
  - 月
- 表示内容:
  - 店名/内容、カテゴリ、日付、ユーザー名、金額
  - 添付画像リンク（別タブ表示）
- 合計表示:
  - 入金を加算、出金を減算した収支合計を表示

### 4.3 サマリータブ
- フィルタ:
  - ユーザー、分類、年、月
- グラフ:
  1. 支出内訳ドーナツ（カテゴリ別）
  2. 入金 vs 出金 バーチャート
- ドーナツ中央テキスト:
  - 対象ユーザー名 + 分類
  - 対象期間（年/月、未指定時は全期間）

## 5. データ仕様

### 5.1 クライアント内主要データ
- `currentType`: `"出金"` または `"入金"`
- `imageBlobs`: 画像のBase64配列（最大3件）
- `allData`: GASから取得した明細一覧
- `userList`: ユーザー一覧（`lineId`, `name`）

### 5.2 明細データ（想定項目）
- `lineId`: LINEユーザーID
- `userName`: 表示名
- `type`: 入金/出金
- `amount`: 金額（数値）
- `category`: カテゴリ
- `shop`: 内容/支払先
- `date`: 日付
- `imageUrls`: 画像URL配列

## 6. APIアクション仕様（フロント利用分）
- `register`
  - 用途: 起動時の自動ユーザー登録
  - 入力: `userId`, `displayName`
- `getUsers`
  - 用途: フィルタ用ユーザー一覧取得
- `getList`
  - 用途: 明細一覧取得
- `upload`
  - 用途: 明細新規登録（画像含む）

## 7. 起動時シーケンス
1. `liff.init()` 実行
2. 未ログイン時は `liff.login()` へ遷移
3. ログイン後 `liff.getProfile()` でユーザー情報取得
4. `register` をGASへ送信（ユーザー登録）
5. フォーム有効化
6. カテゴリ初期化
7. フィルタ初期化（`getUsers`）

## 8. 外部依存
- LIFF SDK
- Chart.js
- Google Apps Script（Webアプリ）
- Googleスプレッドシート
- GitHub / Vercel

## 9. セキュリティ・運用上の注意
- LIFF ID・GAS URLはフロント公開情報となるため、GAS側で入力検証・アクセス制御を実施すること
- スプレッドシートへの直接公開アクセスは避け、GASで権限管理を行うこと
- 画像データ送信量が大きい場合は、サイズ制限や圧縮処理を検討すること
- 本番運用ではエラー監視（GAS実行ログ、Vercelログ）を定期確認すること

## 10. デプロイ運用フロー（推奨）
1. ローカルで修正
2. GitHubへコミット・push（`main`）
3. Vercelが自動デプロイ
4. LINE LIFF画面で動作確認
5. GAS/スプレッドシート側データ整合性を確認
