# ダイエットアプリ 連携仕様書

## 1. 文書概要
- 文書名: ダイエットアプリ 連携仕様書
- 対象システム: 健康管理アプリ（LIFF Webアプリ）
- 目的: Vercel / GitHub / LINE / Googleスプレッドシート / GAS の連携構成と、主要機能・データフローを明文化する
- 作成日: 2026-04-25

## 2. システム構成（全体像）
- フロントエンド: `index.html`（単一HTML + JavaScript）
- 配信基盤: Vercel（GitHub連携デプロイ）
- 認証基盤: LINE LIFF（LINEログイン・プロフィール取得）
- バックエンドAPI: Google Apps Script（Webアプリ公開URL）
- データストア: Googleスプレッドシート
- ソース管理: GitHub リポジトリ `futurewithai24/My-diet-App`（想定）

## 3. 連携先一覧

### 3.1 GitHub
- 用途: ソースコード管理、変更履歴管理、Vercelデプロイ元
- リポジトリ: [futurewithai24/My-diet-App](https://github.com/futurewithai24/My-diet-App.git)（想定）
- 運用想定:
  - `main` ブランチへの push をトリガーにVercel再デプロイ
  - 変更はコミット単位で管理

### 3.2 Vercel
- 用途: 静的フロントエンド配信
- 想定公開URL: `https://my-diet-app.vercel.app`（仮）
- 役割:
  - `index.html` のホスティング
  - LINEアプリ内ブラウザ / 通常ブラウザからのアクセス受付

### 3.3 LINE（LIFF）
- 利用SDK: `https://static.line-scdn.net/liff/edge/2/sdk.js`
- 設定値:
  - LIFF ID: `<YOUR_LIFF_ID>`
- 役割:
  - ログイン状態確認（`liff.isLoggedIn()`）
  - 未ログイン時のログイン誘導（`liff.login()`）
  - ユーザープロフィール取得（`liff.getProfile()`）
    - `userId`
    - `displayName`

### 3.4 Google Apps Script（GAS）
- APIエンドポイント:
  - `https://script.google.com/macros/s/<YOUR_DEPLOYMENT_ID>/exec`
- 通信方式:
  - `POST`
  - Body: `{"action": "...", "data": {...}}`
- フロント側共通関数:
  - `callGas(action, data)`

### 3.5 Googleスプレッドシート
- 用途: 健康記録、ユーザー情報、AI分析履歴の永続化
- スプレッドシートURL: [ダイエットアプリ管理シート](https://docs.google.com/spreadsheets/d/1BrGWRJTnBUBjLOZd3U63ThtDqt_nBdex5FbYkFjrbrI/edit?gid=0#gid=0)
- スプレッドシートID: `1BrGWRJTnBUBjLOZd3U63ThtDqt_nBdex5FbYkFjrbrI`
- 連携方法: GAS経由で読み書き
- 備考: フロントはスプレッドシートへ直接アクセスしない（必ずGAS API経由）

### 3.6 Google Drive（写真保管）
- 用途: 食事写真・体重計画面キャプチャ等の画像保管先
- フォルダURL: [写真保存フォルダ](https://drive.google.com/drive/folders/1MRTwf4J2X2udXR3RhMpC7Gt7vRtBvOJh)
- フォルダID: `1MRTwf4J2X2udXR3RhMpC7Gt7vRtBvOJh`
- 連携方法: GASでBase64画像を受信し、Driveへ保存後URLをスプレッドシートへ記録
- 備考: フォルダ共有設定は「リンクを知っている全員」にせず、原則GAS実行ユーザーのみアクセス可能とする

## 4. 機能仕様（画面別）

### 4.1 記録タブ
- 記録種別:
  - 食事 / 間食 / プロテイン / 体重計データ / 運動 / 体調（お通じ）
- 入力項目（共通）:
  - 記録日
  - メモ
- 入力項目（種別別）:
  - 食事系: タイミング（朝/昼/夜/間食）、写真（最大3枚）
  - 体重計: 体重、体脂肪率、筋肉量（任意）、基礎代謝（任意）
  - 運動: 種目（ジム/散歩など）、時間（分）、消費カロリー（任意）
  - 体調: お通じ有無（あり/なし）、体調メモ
- 写真アップロード:
  - カメラ撮影 or ギャラリー選択
  - Base64化して `imageBlobs` 配列へ保持（最大3件）
- 登録処理:
  - `action: "uploadRecord"` でGASへ送信
  - 成功時: 完了アラート後に画面再読み込み、または一覧再取得
- カロリー推定:
  - 食事写真アップロード後に「写真からカロリー推定」を実行
  - `action: "estimateMealCalories"` でGASへ送信
  - 返却値: 推定カロリー、PFC（たんぱく質/脂質/炭水化物）、推定根拠

### 4.2 履歴タブ
- フィルタ:
  - ユーザー（全員/個別）
  - 記録種別
  - 年
  - 月
- 表示内容:
  - 日付、記録種別、主要値（例: 体重、運動時間）、メモ、ユーザー名
  - 添付画像リンク（別タブ表示）
- 集計表示:
  - 当月の運動回数
  - 当月の体重推移（初日比）
  - お通じ記録率（入力日数ベース）

### 4.3 サマリータブ
- フィルタ:
  - ユーザー、記録種別、年、月
- グラフ:
  1. 体重推移ラインチャート（日次）
  2. 運動時間バーチャート（日次/週次）
  3. 記録種別割合ドーナツ（食事/運動/体重計/体調）
- サマリー表示:
  - 対象ユーザー名
  - 対象期間（年/月、未指定時は全期間）
  - 主要指標（体重増減、運動時間合計、お通じ有無の回数）

### 4.4 AIアドバイスタブ
- 実行方法:
  - 「AI分析を実行」ボタン押下で分析リクエスト送信
- 分析対象:
  - 直近N日（例: 7日/30日）の食事・運動・体重・体調データ
- 返却内容:
  - 総評（現在の傾向）
  - 食事アドバイス
  - 運動アドバイス
  - 翌日の行動提案（3件）
- 実行処理:
  - `action: "analyzeHealth"` をGASへ送信
  - GAS側でAI APIを呼び出して結果を整形して返却

## 5. データ仕様

### 5.1 クライアント内主要データ
- `recordType`: 現在の記録種別
- `imageBlobs`: 画像のBase64配列（最大3件）
- `allRecords`: GASから取得した記録一覧
- `userList`: ユーザー一覧（`lineId`, `name`）
- `aiAdvice`: AI分析結果テキスト

### 5.2 健康記録データ（想定項目）
- `lineId`: LINEユーザーID
- `userName`: 表示名
- `recordType`: 食事/間食/プロテイン/体重計/運動/体調
- `recordDate`: 記録日
- `weight`: 体重（数値, 任意）
- `bodyFat`: 体脂肪率（数値, 任意）
- `exerciseType`: 運動種別（任意）
- `exerciseMinutes`: 運動時間（数値, 任意）
- `bowelMovement`: お通じ有無（あり/なし, 任意）
- `memo`: メモ
- `estimatedCalories`: 推定カロリー（数値, 任意）
- `protein`: たんぱく質（g, 任意）
- `fat`: 脂質（g, 任意）
- `carbohydrate`: 炭水化物（g, 任意）
- `calorieBasis`: 推定根拠テキスト（任意）
- `imageUrls`: 画像URL配列

### 5.3 AI分析履歴データ（想定項目）
- `lineId`: LINEユーザーID
- `period`: 分析対象期間（例: 直近7日）
- `adviceText`: AI返却テキスト
- `createdAt`: 生成日時

## 6. APIアクション仕様（フロント利用分）
- `register`
  - 用途: 起動時の自動ユーザー登録
  - 入力: `userId`, `displayName`
- `getUsers`
  - 用途: フィルタ用ユーザー一覧取得
- `getRecords`
  - 用途: 健康記録一覧取得
- `uploadRecord`
  - 用途: 健康記録の新規登録（画像含む）
- `analyzeHealth`
  - 用途: 記録データを基にAIアドバイス生成
- `estimateMealCalories`
  - 用途: 食事写真とメモからカロリー/PFCを推定
  - 出力: `estimatedCalories`, `protein`, `fat`, `carbohydrate`, `basis`, `referenceSources`

## 7. 起動時シーケンス
1. `liff.init()` 実行
2. 未ログイン時は `liff.login()` へ遷移
3. ログイン後 `liff.getProfile()` でユーザー情報取得
4. `register` をGASへ送信（ユーザー登録）
5. フォーム有効化
6. 記録種別・入力欄の初期化
7. フィルタ初期化（`getUsers` / `getRecords`）

## 9. セキュリティ・運用上の注意
- LIFF ID・GAS URLはフロント公開情報となるため、GAS側で入力検証・アクセス制御を実施すること
- スプレッドシートへの直接公開アクセスは避け、GASで権限管理を行うこと
- 画像データ送信量が大きいため、サイズ制限・圧縮・枚数上限を必須化すること
- AI APIキーはフロントに置かず、必ずGAS側のプロパティで安全に管理すること
- 本番運用ではエラー監視（GAS実行ログ、Vercelログ）を定期確認すること
